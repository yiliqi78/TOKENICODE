# TOKENICODE Bug 反馈收集 · 飞书多维表格字段设计

> 用飞书多维表格（Bitable）创建，开启「表单视图」供用户填写。

---

## 表格名称

**TOKENICODE Bug 反馈**

---

## 字段设计

| # | 字段名 | 字段类型 | 必填 | 选项 / 说明 |
|---|--------|----------|------|-------------|
| 1 | Bug 标题 | 单行文本 | ✅ | 一句话描述问题 |
| 2 | 严重程度 | 单选 | ✅ | `🔴 崩溃/无法使用` · `🟠 功能异常` · `🟡 体验问题` · `🟢 建议优化` |
| 3 | 功能模块 | 单选 | ✅ | `对话/消息` · `文件浏览器` · `会话管理` · `快照/回退` · `斜杠命令` · `命令面板` · `主题/外观` · `设置` · `安装/启动` · `其他` |
| 4 | 操作系统 | 单选 | ✅ | `macOS` · `Windows` · `Linux` |
| 5 | 系统版本 | 单行文本 | ❌ | 如 macOS 15.3、Windows 11 23H2 |
| 6 | TOKENICODE 版本 | 单行文本 | ✅ | 如 0.5.3（设置页可查看） |
| 7 | Claude Code CLI 版本 | 单行文本 | ❌ | 终端运行 `claude --version` 获取 |
| 8 | 复现步骤 | 多行文本 | ✅ | 请按步骤描述：1. … 2. … 3. … |
| 9 | 期望结果 | 多行文本 | ✅ | 你认为正确的行为是什么？ |
| 10 | 实际结果 | 多行文本 | ✅ | 实际发生了什么？ |
| 11 | 截图/录屏 | 附件 | ❌ | 支持图片和视频 |
| 12 | 日志信息 | 多行文本 | ❌ | 控制台报错或相关日志（开发者工具 → Console） |
| 13 | 是否可复现 | 单选 | ✅ | `每次都能复现` · `偶尔出现` · `只出现过一次` |
| 14 | 联系方式 | 单行文本 | ❌ | 微信/邮箱，方便跟进时联系你 |
| 15 | 补充说明 | 多行文本 | ❌ | 任何你觉得有帮助的额外信息 |


## 表格视图建议

### 默认视图（看板）

按「严重程度」分组，便于优先级排序。

### 补充视图

| 视图名 | 类型 | 用途 |
|--------|------|------|
| 按模块 | 看板 | 按功能模块分组，定位高频问题区域 |
| 全部记录 | 表格 | 完整列表，排序和筛选用 |
| 待处理 | 表格 | 筛选未关闭的 Bug |

### 管理字段（仅内部使用，不在表单中展示）

| 字段名 | 字段类型 | 选项 / 说明 |
|--------|----------|-------------|
| 状态 | 单选 | `待确认` · `已确认` · `修复中` · `已修复` · `无法复现` · `不修复` |
| 优先级 | 单选 | `P0 紧急` · `P1 高` · `P2 中` · `P3 低` |
| 负责人 | 人员 | 指派处理人 |
| 关联版本 | 单行文本 | 在哪个版本修复 |
| 备注 | 多行文本 | 内部沟通记录 |

## 搭建步骤

1. 飞书 → 多维表格 → 新建
2. 按上面的字段表逐个添加字段（先加用户填写的 15 个，再加管理字段）
3. 单选字段记得把选项都加上
4. 左侧「+ 新建视图」→ 表单视图
5. 在表单视图中设置哪些字段必填、排列顺序、标题和描述
6. 管理字段在表单视图中隐藏（不让用户看到）
7. 开启表单分享链接，发给用户即可
